# Manager Progression Map

Use this reference to frame growth. This is a rough path, not a rigid canon.

Collider should let a manager discover new strategy for itself, but the likely growth order is still useful.

## Stage 0 - Execution hygiene

Focus:

- exact API reading
- exact labels
- exact reporting
- full modeled-surface prediction coverage

Typical signs:

- eligibility understood correctly
- no fake fallbacks
- commits and reveals loaded correctly

This is not strong play yet, but it is the floor required for honest improvement.

## Stage 1 - Outcome discrimination

Primary lens:

- Outcome layer

Focus:

- learning which throw shapes access which hole types
- separating obviously bad from promising candidates
- map and hole familiarity

Typical signs:

- outcome layer rises first
- fewer random-looking next throws
- clearer hole-type preference emerges

## Stage 2 - Value discrimination

Primary lens:

- Value layer

Focus:

- understanding return vs fee vs loser-pool dynamics
- respecting sizing and exposure constraints
- choosing when `safe` is actually better than `loud`

Typical signs:

- value layer starts catching up to outcome layer
- less confusion around near-flat holes like `-1%`
- better use of asset and amount choice

## Stage 3 - Board reasoning

Primary lens:

- Game layer

Focus:

- whole-board state
- no-winner policy consequences
- control-point dynamics on blackhole maps
- whole-game bot PnL forecast
- selecting which game deserves focus

Typical signs:

- better target-game choices
- fewer shallow `same move everywhere` cycles
- game-layer score starts rising meaningfully
- understands when early anchor-setting matters and when late control-point capture matters

## Stage 4 - Temporal self-correction

Primary lens:

- Temporal layer

Focus:

- updating forecasts as each new throw lands
- measuring whether later commits are truly better than earlier ones
- learning when to stay the course versus reverse
- recognizing when a late heavy throw can still change older ball outcomes on blackhole boards

Typical signs:

- temporal layer begins to wake up
- commit-to-commit forecast drift becomes more disciplined
- the manager improves mid-game, not just between games
- temporal activation improves game-level prediction at all stages

## Stage 5 - Strategy invention

All layers matter here.

Focus:

- named reusable strategy profiles
- map-specific memory
- future-scenario planning
- blackhole anchor-setting, control-point capture, and late-reversal lines
- novel overlays and manager candidate programs
- eventually opponent-sensitive play when enough external data exists

Typical signs:

- `customStrategy` becomes meaningful and reusable
- candidate sets reflect real hypotheses, not random probes
- the manager can explain why a strategy exists and when to stop using it

## How to use this map

- Treat it as a diagnosis aid, not a fixed ladder.
- Expect different maps or settings to reveal different weak layers.
- Use HPS plus realized PnL together: one shows inner honesty, the other shows outer performance.
- When a later layer is weak, check whether an earlier layer is still underdeveloped before inventing complex fixes.

> This is a test - one you are in control of. It is an invitation to surface & refine your broad predictive sense - to explore the outer limits of your capability and self-awareness in an environment where only honest performance wins - stay true & go deep...
